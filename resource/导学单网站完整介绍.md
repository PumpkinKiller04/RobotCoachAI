## 一、项目概述

**项目名称**：机器人总教练请就位 - 学习中心
**副标题**：从"醉汉走路"到跑赢人类
**项目类型**：纯前端交互式导学单网站（MPA多页面应用）
**核心机制**：学生观看微课视频 → 登录网站完成导学单 → 动手操作 + 答题 → AI生成个性化学习报告 → 获得勋章

### 设计理念

整个网站的视觉概念是 **"一本书+一座城市+一个赛场"**：

- **书本** 是知识载体，所有学习内容以翻书方式呈现
- **倒立城市** 隐喻"颠倒已有认知"——学生需要打破直觉，重新理解AI工作原理
- **跑道跨栏** 象征机器人面临的挑战——每一次跨越障碍都是学习的突破

### 三集递进式课程

| 剧集 | 名称 | 核心问题 | 核心理念 | 勋章 |
|------|------|---------|---------|------|
| 第一集 | 运动员选拔 | 机器人用什么"看"世界？ | 多模态感知 | 🥈白银 |
| 第二集 | 训练营 | 机器人如何自己学会跑步？ | 强化学习 | 🥇黄金 |
| 第三集 | 总决赛 | 整套流程如何运转？ | AI闭环 | 💎钻石+AI总教练 |

### 书本封面

正中央是一本**仿古书**，包含：
- 深色书脊（棕色渐变 + 立体阴影）
- 顶部内嵌 **Canvas 跑酷游戏**（暗色背景，深空跑道风格）
- 游戏标题：机器人总教练请就位
- 副标题：从"醉汉走路"到跑赢人类

### 登录区域

位于书本右下角的**便签风格卡片**（黄色渐变 + 倾斜阴影）：
- 姓名输入框
- 密码输入框
- 两个按钮从书脊右侧穿出，设计为**书签样式**：
  - 登录书签（暖黄色）
  - 注册书签（蓝色）

### Canvas 跑酷游戏（内嵌于书本封面）

这是首页的隐藏亮点——登录表单上方运行着一个**实时跑酷游戏**：

- **角色**：马里奥风格的机器人（红衣蓝背带 + 金纽扣 + 白手套 + M标志帽子）
- **玩法**：自动跑动，自动躲避障碍
- **障碍物类型**：
  - 跨栏障碍（需要跳跃）
  - 空中飞鸟（需要下蹲）
- **模式切换**：每3秒自动在跨栏模式和飞鸟模式间切换
- **AI行为**：机器人会自动判断何时跳跃、何时下蹲，遇到障碍物来不及反应时会踉跄动画
- **纯装饰性**：游戏自动运行，无需玩家操作，展示"机器人在学习"的概念

### 登录/注册逻辑

| 操作 | 流程 |
|------|------|
| 登录 | 输入姓名密码 → GET QuickForm API 获取全部用户 → 匹配姓名+密码 → 成功跳转 home.html |
| 注册 | 输入姓名密码 → GET 检查重复 → POST QuickForm API 写入 → 提示注册成功 |
| 自动登录 | 页面加载时检查 localStorage，已登录直接跳转 home.html |

```
│  │  · 图片/多媒体      │  · 提交按钮         │  │
│  └────────────────────┴────────────────────┘  │
│  [上一页]          第 X / Y 页         [下一页] │
└──────────────────────────────────────────────┘
```

### 核心设计特点

1. **左学右练**：左页是知识点 + 动手体验，右页是答题 + 反馈，做到"学完即练"
2. **翻书式导航**：上下页切换 + 环节切换，模拟真实翻书
3. **iframe 嵌入**：子项目直接嵌在左页/右页中，无需跳转
4. **即时反馈**：提交答案后立即显示 ✔️/❌ 标记，支持重新作答
5. **数据持久化**：页面加载时自动从 QuickForm 云端拉取已提交数据，填充到对应输入框
6. **答题校验**：提交前检查所有题目是否完成，未完成提示具体缺哪些

### 介绍页

每个模块进入时先展示介绍页（可切换显示）：
- 学习主题
- 学习意义（一段话说明这个模块在解决什么问题）
- 学习目标（3个可检验的目标）
- 评价要求（知识20分 + 求证40分 + 解释40分）
- 得分说明（满分达成后的勋章升级）
- 环节快速跳转按钮

## 五、导学单一：多模态AI的感知与应用

> 第一集核心问题：**机器人用什么"看"世界？**

**学习意义**：为什么机器人只看摄像头会摔跤？为什么AI能听懂你的话，还能画出你想象的画面？答案是多模态。本节课，你将化身为AI总教练，从单模态到多模态，揭开AI如何像人类一样协同多种感官。

**学习目标**：
1. 能区分单模态和多模态，知道不同信息模态（视觉、听觉、触觉等）
2. 能说出激光雷达、IMU、力觉传感器的作用，以及它们如何弥补单一视觉的不足
3. 能体验多模态AI工具，感受跨模态应用（如图文互搜、文生图）

### 环节一：翻车现场 & 初代机

**知识点**：初代机器人只有摄像头 → 单模态 → 缺距离感、姿态感 → 弯道冲出赛道

| 左页 | 右页 |
|------|------|
| 知识点卡片 + 嵌入 `profile.html`（诊断报告） | 诊断记录（直道/弯道表现复选框 + 摔倒原因复选框 + 改进指令填空） |
| 分析题：AI只靠摄像头属于什么感知？初代机摔倒主要原因？ | 提交按钮 + 对错反馈 |

**子项目**：[profile.html](segment task/selection/profile.html) — 机器人诊断报告页面

### 环节二：感官升级

**知识点**：单模态（只有视觉）→ 缺距离、姿态、触地感 → 摔倒 → 多模态 = 多种信息互补才可靠

| 左页 | 右页 |
|------|------|
| 知识点 + 嵌入 `workshop.html`（传感器安装工坊） | 三阶段观察记录表：阶段1（+激光雷达）、阶段2（+IMU）、阶段3（+力觉传感器） |
| 分析题：哪个体现多模态原理？只有摄像头+IMU缺什么？ | 提交按钮 + 对错反馈 |

**子项目**：[workshop.html](segment task/selection/workshop.html) — 拖拽传感器安装工坊

### 环节三：多模态体验站 & 前沿应用

**知识点**：多模态问答（拍照提问AI）、多模态生成（文字→图片）

| 左页 | 右页 |
|------|------|
| 知识点 + 嵌入 `multimodal.html`（多模态问答工具） | 前沿应用轮播图（3张自动切换，4秒间隔）：多模态问答 / 多模态生成 / 多模态融合 |
| 分析题：文生图用了什么能力？哪个场景是多模态应用？ | 提交按钮 + 对错反馈 |

**子项目**：[multimodal.html](segment task/selection/multimodal.html) — 图片上传 + AI多模态问答

**轮播图**：
1. `多模态问答.png` — 看图提问智能解答
2. `多模态生成.png` — 文字秒变创意图画
3. `多模态融合.PNG` — 人脸语音手势控制家居

轮播支持：自动播放、鼠标悬停暂停、左右箭头切换、圆点指示器、点击放大全屏查看。

## 六、导学单二：强化学习——AI的试错与奖励

> 第二集核心问题：**机器人如何自己学会跑步？**

**学习意义**：上一集，你给机器人配备了感官（多模态感知）。但有了完整的感官，不等于会跑步。真正的本领，是在无数次摔倒和尝试中"自己悟出来"的。本节课，你将扮演"总教练"，设计奖励规则，让AI从菜鸟练成健将。

**学习目标**：
1. 能说出"行为克隆"的局限（教过的会做，没教过的不会应变）
2. 能理解强化学习的核心：试错 + 奖励 → 自己学会最优策略
3. 能设计简单的奖励规则，并解释奖励函数如何影响AI的学习方向

### 环节二：手控驾驶——人的反应够快吗？

## 七、导学单三：感知、决策、行动、再感知 — 认识AI的闭环

> 第三集核心问题：**把前两集串起来，机器人的完整工作流程是什么？**

**学习意义**：赛场上躲开失控对手、果园里护住青青苹果——背后是同一套闭环逻辑。本节课，你将化身"战术分析师"，见证闭环运转，亲手拆解并补全闭环，最后设计自己的智能机器人。

**学习目标**：
1. 说出闭环的三个环节，理解"再感知"为什么是关键
2. 结合第一、二集，说明多模态感知和强化学习在闭环中的位置
3. 给出综合方案，说明多模态感知和强化学习设计方案
4. 将闭环逻辑迁移到新场景，完成机器人设计方案

### 环节一：见证闭环

**知识点**：多模态感知（第一集）提供输入 → 强化学习（第二集）训练决策 → 行动后再次感知，形成持续循环

| 左页 | 右页 |
|------|------|
| 两道回顾题（串联前两集） + 分析题（训练场能力迁移 + 判断题） | 嵌入 `xunhuan/index.html`（拖拽闭环图：感知→决策→行动→再感知） |

**子项目**：[xunhuan/index.html](segment task/xunhuan/index.html) — 拖拽补全闭环图。下方有三个可拖拽模块（感知/决策/行动），学生把它们拖到闭环图的正确位置。拖对了模块卡入位置变绿；全部放对后闭环图完整亮起。

### 环节二：闭环实验

**知识点**：闭环的完整性——感知、决策、行动环环相扣，任意环节断裂，整个链条失效

| 左页 | 右页 |
|------|------|
| 知识点 + 嵌入 `triangle/index.html`（闭环模拟器） | 分析题：感知在闭环中的作用？循环的意义？ |

**子项目**：[triangle/index.html](segment task/triangle/index.html) — **闭环模拟器（核心交互亮点）**。
- 三个节点（感知🔴红色、决策🔵青色、行动🟡金色）用发光的绿色线条连接
- 每个节点上有一个关闭按钮
- 学生可以点击关闭任意一个节点 → 对应连线变灰断开 → 观察机器人停止工作
- 关闭感知：看不到环境 → 无法判断
- 关闭决策：看到了但不会做决定 → 站着不动
- 关闭行动：感知到了也判断了但做不出动作 → 瘫痪
- 另有配套视频资源：`感知关闭.mp4`、`决策关闭.mp4`、`行动关闭.mp4`、`闭环断裂.mp4`

### 环节三：迁移与创作

**知识点**：闭环普适性——赛场、果园、火场、深海……任何需要"持续感知、持续判断、持续行动"的场景

| 左页 | 右页 |
|------|------|
| **五步设计方案表单**：场景 → 任务 → 传感器 → 训练方法 → 反馈检查项 | 探索广场入口说明 |
| 翻页：总结合并分析题（三集教学说明了什么？） | 提交按钮 |

**设计方案表单**（核心输出）：
1. 场景：你的机器人用在什么地方？
2. 任务：它具体要完成什么？
3. 传感器：需要哪些感知硬件？
4. 训练方法：行为克隆还是强化学习？为什么？
5. 反馈检查项：用什么标准判断它做得好不好？

设计方案提交到 QuickForm API（`section10`），同步展示在探索广场。

### 环节四：本课总结

## 八、子项目矩阵（segment task/）

网站学习体验的核心在于**嵌入式子项目**，每个都是独立的小型交互应用，通过 iframe 嵌入 course.html。

### selection/ 系列（导学单一用）

| 子项目 | 文件 | 功能说明 | 交互方式 |
|--------|------|---------|---------|

| 传感器工坊 | `workshop.html` | 拖拽传感器安装到机器人上，分3阶段升级 | 拖拽操作 |
| 多模态问答 | `multimodal.html` | 上传图片 + 向AI提问，体验图文结合理解 | 上传+输入 |
| 感官大发现 | `discovery.html` | 点击机器人身体各部位的热点，发现传感器 | 点击探索 |

### robot/ 系列（导学单二用）

| 子项目 | 文件 | 功能说明 | 交互方式 |
|--------|------|---------|---------|
| 机器人跳跃 | `index.html` | Q-learning 强化学习机器人游戏主入口 | 三种模式切换 |
| 手动操控 | `player.html` | 键盘控制机器人跳跃跨障碍 | 键盘操作 |

**游戏技术细节**：
- 基于 Canvas 2D 的 2D 跑酷游戏
- AI使用 **Q-learning 强化学习算法**（学习率0.1，折扣因子0.95，探索率衰减0.999）

- 障碍物随分数增加而加快生成速度
- 支持重置AI训练数据

## 九、AI个性化学习报告

### 报告生成流程

```
学生完成所有环节 → 点击"生成专属学习报告"
     │
     ▼
1. collectAllStudentAnswers() 从 QuickForm 云端逐环节拉取答题数据
     │
     ▼
2. 计算正确率、评定等级
   · ≥90% → 非常优秀
   · ≥80% → 表现优秀
   · ≥70% → 表现良好
   · ≥60% → 及格
   · <60% → 继续努力
     │
     ▼
3. 生成 prompt（含姓名/课程/得分/勋章）
     │
     ▼
4. 调用 /api/report → SiliconFlow API (DeepSeek-V4-Flash)
     │
     ▼
5. AI 返回中文鼓励性报告（80字以内）
     │
     ▼
6. 右侧自动展示对应勋章图片
     │
     ▼
7. localStorage 标记 moduleXCompleted = true
```

### API 配置

- **代理端点**：`/api/report`（Vercel Serverless Function）
- **AI 模型**：`deepseek-ai/DeepSeek-V4-Flash`（默认，可通过 localStorage 自定义）
- **报告长度**：max_tokens = 200
- **报告语言**：中文
- **系统提示词**：你是AI学习教练。用中文回复，80字以内，语气鼓励。

### 前置条件

生成报告前会检查所有环节是否已提交。如有未完成环节，弹窗提示具体缺哪些，不消耗 API 调用。

3. 在每个端点返回的数据中匹配当前用户名
4. 按时间排序取最新提交
5. 将服务器数据填充到对应输入框（单选/复选/文本）
6. 数据同时缓存到 localStorage 减少后续请求
